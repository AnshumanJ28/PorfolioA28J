01 — Overview

About

Most of my portfolio is one long argument: a model isn't finished when the notebook runs cleanly. It's finished when it's versioned, monitored, tested, and still working after the data shifts underneath it. That's the thread running through my MLOps pipeline, my drift-monitored forecasting work, and my digital twin project — production concerns first, novelty second.

The one exception is AlphaZ0, my self-play chess engine — built purely because chess is the thing I'd think about anyway, so I built the AI to think about it with me. Everything else on this page exists because I wanted to know how ML systems actually break in production, and the only way to find out was to build enough of them.

When I'm not training something, I'm probably deep in a LeetCode session in C++, or losing track of time on a chessboard.
